# 🔔 Push-уведомления в Jetesk

## Обзор

В мессенджер Jetesk добавлена поддержка **Web Push уведомлений** для уведомления пользователей о новых сообщениях даже когда браузер закрыт.

## Что добавлено

### Серверная часть (`messenger_server.py`)

1. **Модель БД `PushSubscription`**
   - Хранит подписки пользователей на push-уведомления
   - `endpoint` - URL push-сервиса браузера
   - `p256dh` - публичный ключ шифрования
   - `auth` - секрет аутентификации

2. **VAPID ключи**
   - Автоматически генерируются при первом запуске
   - Используются для авторизации push-уведомлений
   - Можно задать через переменные окружения:
     - `VAPID_PUBLIC_KEY`
     - `VAPID_PRIVATE_KEY`

3. **API endpoints**
   - `GET /api/push/public-key` - получить публичный VAPID ключ
   - `POST /api/push/subscribe` - подписаться на уведомления
   - `POST /api/push/unsubscribe` - отписаться от уведомлений
   - `GET /api/push/status` - проверить статус подписки

4. **Функция `send_push_notification()`**
   - Отправляет push-уведомление пользователю
   - Автоматически вызывается при новых сообщениях
   - Удаляет невалидные подписки (ошибка 410)

### Клиентская часть (`index.html`)

1. **Service Worker (`sw.js`)**
   - Обрабатывает push-события
   - Показывает уведомления
   - Обрабатывает клики по уведомлениям

2. **JavaScript функции**
   - `initPushNotifications()` - инициализация
   - `subscribeToPush()` - подписка
   - `unsubscribeFromPush()` - отписка
   - `updatePushStatus()` - обновление UI

3. **UI в настройках**
   - Кнопка включения/выключения уведомлений
   - Индикатор статуса

### Файлы

```
product/
├── messenger_server.py    # Сервер с push-интеграцией
├── index.html             # Клиент с push-кодом
├── sw.js                  # Service Worker
└── requirements.txt       # Зависимости (pywebpush, cryptography)
```

## Установка

### 1. Установите зависимости

```bash
cd product
pip install -r requirements.txt
```

### 2. Запустите сервер

```bash
python messenger_server.py
```

При первом запуске будут сгенерированы VAPID ключи:
```
=== VAPID ключи сгенерированы ===
Public Key: BEl...
Private Key: 3aF...
```

### 3. Для продакшена (Vercel)

Добавьте переменные окружения в Vercel:
- `VAPID_PUBLIC_KEY` - публичный ключ
- `VAPID_PRIVATE_KEY` - приватный ключ

**Важно:** Сохраните ключи после первой генерации!

## Как это работает

### Подписка на уведомления

1. Пользователь входит в аккаунт
2. Service Worker регистрируется в браузере
3. При открытии настроек показывается кнопка "Включить уведомления"
4. При клике браузер запрашивает разрешение
5. После разрешения создаётся подписка
6. Подписка отправляется на сервер

### Получение уведомлений

1. Пользователь А отправляет сообщение пользователю Б
2. Сервер вызывает `send_push_notification()` для получателя
3. Push-сервис браузера получает уведомление
4. Service Worker показывает системное уведомление
5. При клике открывается чат с отправителем

## Требования

- **HTTPS** (на Vercel уже есть)
- **Современный браузер** (Chrome, Firefox, Edge, Safari 16+)
- **Service Worker поддержка**

2. Войдите под двумя разными пользователями в разных браузерах
3. Включите push-уведомления в настройках
4. Отправьте сообщение
5. Проверьте получение уведомления

### На продакшене

1. Задеплойте на Vercel
2. Откройте сайт в браузере
3. Включите уведомления
4. Протестируйте с другим пользователем

## Отладка

Откройте консоль разработчика (F12) и следите за логами:
- `[Push]` - логи push-клиента
- `Push:` - логи push-сервера

### Частые проблемы

**"Push notifications not supported"**
- Браузер не поддерживает Service Worker
- Нет HTTPS (кроме localhost)

**"Permission denied"**
- Пользователь запретил уведомления
- Нужно очистить разрешения в настройках браузера

**"410 Gone"**
- Подписка невалидна
- Сервер автоматически удалит её

## Переменные окружения

```bash
# VAPID ключи (сохраните после первой генерации!)
VAPID_PUBLIC_KEY=BEl...
VAPID_PRIVATE_KEY=3aF...

# Для Vercel добавьте в Settings → Environment Variables
```

## Безопасность

- VAPID ключи уникальны для каждого сервера
- Подписки зашифрованы
- Push отправляются только авторизованным пользователям
- Ключи не должны коммититься в Git

## Отключение

Чтобы отключить push-уведомления:
1. Откройте настройки
2. Нажмите "Выключить уведомления"
3. Или очистите данные сайта в браузере
